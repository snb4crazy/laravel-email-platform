# Test Environment Configuration Guide

## Overview

Your application now has proper test configuration set up. Here's what was configured and why.

## Configuration Files

### 1. `.env` - Application Environment Variables

**Key settings for testing:**

```env
APP_ENV=local              # Enables debug mode in development
APP_URL=http://local.eplatform.com  # Used for URL generation (now with protocol!)
APP_DEBUG=false            # Set to true for development to see detailed errors
```

**For local development:**
```env
APP_DEBUG=true             # Shows detailed error pages
APP_ENV=local              # Uses development settings
```

**For production:**
```env
APP_DEBUG=false            # Never show detailed errors in production
APP_ENV=production         # Uses production settings
```

### 2. `phpunit.xml` - PHPUnit Test Configuration

**Test-specific environment overrides:**

```xml
<env name="APP_ENV" value="testing"/>
<env name="APP_URL" value="http://localhost"/>
<env name="DB_CONNECTION" value="sqlite"/>
<env name="DB_DATABASE" value=":memory:"/>
<env name="MAIL_MAILER" value="array"/>
<env name="QUEUE_CONNECTION" value="sync"/>
<env name="CACHE_STORE" value="array"/>
```

**Why these settings for tests:**

| Setting | Value | Reason |
|---------|-------|--------|
| APP_ENV | testing | Tells Laravel this is a test environment |
| APP_URL | http://localhost | Isolated test URL, not your actual domain |
| DB_CONNECTION | sqlite | Fast, file-based database |
| DB_DATABASE | :memory: | In-memory database (no files, super fast) |
| MAIL_MAILER | array | Captures emails in memory instead of sending |
| QUEUE_CONNECTION | sync | Jobs execute immediately (easier to test) |
| CACHE_STORE | array | In-memory cache for tests |
| SESSION_DRIVER | array | In-memory sessions for tests |

## Understanding APP_URL

### What is APP_URL?

`APP_URL` tells Laravel the base URL of your application. It's used for:

1. **URL Generation**: `url()` and `route()` helpers
2. **Email Links**: Links in emails generated by your app
3. **Redirects**: Where to redirect users
4. **Tests**: How the test HTTP client makes requests

### Format

```
Scheme://Domain:Port/Path

http://local.eplatform.com
https://api.yourdomain.com:443
http://localhost:8000
```

**Must include:**
- ✅ Scheme (`http://` or `https://`)
- ✅ Domain or IP address
- ⚠️ Port (optional, uses default 80 for http, 443 for https)
- ⚠️ Path (optional, usually empty for API root)

### Common Mistakes

❌ `local.eplatform.com` - Missing protocol (WRONG)
✅ `http://local.eplatform.com` - Correct format

❌ `http://` - Missing domain (WRONG)
✅ `http://localhost` - Correct format

❌ `localhost:8000` - Missing protocol (WRONG)
✅ `http://localhost:8000` - Correct format

## Environment Override Hierarchy

When `php artisan test` runs:

1. **Start with**: `.env` file
2. **Override with**: `phpunit.xml` environment variables
3. **Final**: Test environment uses phpunit.xml settings

This means you can have different configurations for development and testing without modifying code.

## Common Test Issues & Solutions

### Tests Getting 404

**Cause**: APP_URL missing protocol

**Solution**: Update `.env` to include `http://` or `https://`

```env
# Before
APP_URL=localhost

# After
APP_URL=http://localhost
```

### Tests Can't Connect to Database

**Cause**: Wrong DB_CONNECTION in phpunit.xml or test trying to use production DB

**Solution**: Ensure phpunit.xml uses sqlite

```xml
<env name="DB_CONNECTION" value="sqlite"/>
<env name="DB_DATABASE" value=":memory:"/>
```

### Tests Running Slowly

**Cause**: Using redis or real databases instead of in-memory

**Solution**: Check phpunit.xml overrides these:
- `CACHE_STORE=array`
- `QUEUE_CONNECTION=sync`
- `DB_DATABASE=:memory:`

### Emails Being Sent During Tests

**Cause**: phpunit.xml not setting MAIL_MAILER to array

**Solution**: Verify phpunit.xml has:

```xml
<env name="MAIL_MAILER" value="array"/>
```

And code uses `Mail::fake()` if needed.

## Best Practices

### Development

```bash
# Clear caches between major changes
php artisan cache:clear
php artisan config:clear
php artisan route:clear
```

### Before Deploying

```bash
# Run full test suite
php artisan test

# Check for warnings
php artisan config:cache
php artisan route:cache

# Verify database
php artisan migrate:status
php artisan migrate --dry-run
```

### After Deployment

```bash
# Clear application cache on server
php artisan cache:clear
php artisan config:clear

# Run migrations if needed
php artisan migrate

# Start queue worker if not already running
php artisan queue:work
```

## Environment Files Reference

### `.env` - Your Local Configuration
- Not committed to git (in .gitignore)
- Unique to your machine
- Changed for development vs production
- Contains secrets (API keys, database passwords)

### `.env.example` - Template (for git)
- Checked into git
- Shows all available env variables
- Has dummy values
- Documentation of all settings needed

### `phpunit.xml` - Test Configuration
- Checked into git
- Applies only when running tests
- Overrides .env settings for testing
- Safe to share (no secrets, test-only)

### `bootstrap/app.php` - Application Bootstrap
- Checked into git
- Loads environment and configuration
- Not modified usually
